## 1. サービス概要
本アプリは、前職の英会話スクールで感じた「宿題管理の非効率さ」を解決するために開発した、前職の英会話スクール専用の宿題管理アプリです。
- 生徒はいつでも自分の宿題や単語テスト範囲を確認でき、毎週取り組むべき学習内容を自身で管理できます。
- 欠席時でも校舎へ問い合わせることなく必要な情報を確認できるため、学習機会の損失を防ぐことができます。
- 講師側は宿題やテスト範囲をオンライン上で一元管理できるため、印刷業務や問い合わせ対応の負担を軽減できます。
本アプリは、生徒・保護者・講師の三者がより効率的に学習・指導を行える環境の実現を目指して開発しました。

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情
本アプリを開発しようと思ったきっかけは、前職の英会話スクールでの勤務経験です。当時、生徒への宿題管理や単語テストの運用を行う中で、紙ベースの管理に多くの課題があることを実感しました。
1. 宿題内容の修正対応が煩雑

宿題プリントは事前にダブルチェックを行ったうえで配布していましたが、その後にカリキュラム作成者から内容の修正連絡が入ることが頻繁にありました。その場合、講師は授業内で口頭説明を行うか、修正版を再印刷して生徒へ配布しなければなりません。修正の度に追加対応が発生し、講師・生徒双方に負担がかかっていました。
2. 印刷コストと管理コストが発生する

宿題プリントは毎月配信されるカリキュラムに合わせて作成・印刷されます。現場では担当講師が印刷を担当していましたが、印刷枚数が不足すると再度印刷しなければならないため、多めに印刷するケースが少なくありませんでした。また、全16校舎で毎月宿題プリントを印刷していることを考えると、用紙代やインク代などの印刷コストも無視できない課題でした。
3. 欠席した生徒が宿題内容を確認しづらい

生徒が授業を欠席すると、次回までに取り組むべき宿題が分からなくなります。その際、生徒や保護者は校舎へ電話をかけて確認する必要があり、スタッフはクラスファイルを確認して口頭で伝えたり、連絡アプリを通じて宿題内容を送信したりしていました。このような問い合わせは頻繁に発生しており、授業準備や他業務を行っている講師にとって大きな負担となっていました。また、生徒の中には「休んだから宿題はやらなくてもいい」と考え、宿題を行わずに次回の授業へ参加するケースも見られました。
4. 月例単語テストの運用にも課題があった

前職の英会話スクールでは、毎月末に単語テストを実施していました。テスト範囲は担当講師が範囲表を印刷して配布するか、授業内でノートに記入させる形で伝えていました。しかし、宿題管理と同様に、
- テスト範囲の修正が発生する
- 講師が範囲を伝え忘れる
- 生徒が欠席して範囲を把握できない

といった問題が頻繁に起こっていました。その結果、生徒がテスト当日に初めて範囲を知るケースもあり、十分な準備ができないまま受験することもありました。
このように、宿題や単語テストの管理を紙ベースで行う運用には、

- 情報修正時の対応負担
- 印刷コスト
- 欠席者への情報共有
- 講師側の管理工数

といった課題がありました。

そこで私は、「生徒がいつでも最新の宿題やテスト範囲を確認でき、講師側も一元管理できる仕組みがあれば、双方の負担を大きく軽減できるのではないか」と考え、本アプリの開発を決意しました。

---

### 2-2. なぜそれが「気になった」のか
1. 保護者からの宿題確認の問い合わせを減らしたい

英会話スクールでは、宿題に関する問い合わせが日常的に発生します。例えば、
- 欠席したので翌週までの宿題範囲を教えてほしい
- 振替授業に参加するので宿題範囲を確認したい
- 宿題プリントを紛失したため再送してほしい

といった内容です。これらの問い合わせがあるたびに、スタッフはクラスファイルやカリキュラムを確認しなければなりません。また、宿題プリントの再送を依頼された場合は、資料をPDF化し、連絡アプリを通じて送信する必要があります。

2. 「宿題がわからなかったからやらなかった」をなくしたい

スクールでは毎週宿題が出されますが、生徒が欠席した場合、「宿題の内容がわからなかった」という理由で宿題を行わずに次回の授業へ参加するケースが少なくありません。
しかし、宿題は授業で扱う長文読解や文法事項の理解を深めるために作成されています。そのため、宿題を行わないまま授業を受けると、学習内容の定着が不十分になり、授業の理解度にも影響を与えます。

3. 宿題をやることが生徒たちの日々の英語力向上につながる。

私自身の指導経験から、毎週の宿題や単語テストの準備を継続して行う生徒ほど、英検で合格する確率が高い傾向にあります。英語力は短期間で身につくものではなく、日々の積み重ねによって向上します。そのため、生徒が継続的に宿題へ取り組める環境を整えることは非常に重要です。

## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)
### 導入に関して
  1. まずはテスト運用として、スクールに通う一部の生徒に実際に利用してもらい、利用状況の分析や操作性に関するフィードバックの収集を行います。
  2. 収集した意見をもとに機能改善やUI/UXの調整を実施し、より学習しやすいアプリへと改善していく予定です。
  3. その後、全校の生徒を対象に展開し、全16校舎で生徒向けのアプリのプレゼンを行う。

### 本アプリの利用を定着させるためには、
  1. ユーザーである生徒にアプリを通じてどのように学習するかを体験してもらう。
  2. 講師にも実際に使用してもらい、アプリの操作・使用方法についてプレゼンを行う。
  3. 宿題の一つとして、自宅で音読学習を取り入れるように促す。

### 実際に導入してもらうために
実際に導入してもらうために本アプリを実際の英会話スクールに導入してもらうためには、単に機能を提供するだけでなく、運営側が抱える懸念を解消する必要があります。

1. 現場の講師に価値を感じてもらう

どれだけ便利なシステムであっても、現場の講師が必要性を感じなければ導入には至りません。

本アプリでは、

- 宿題確認の電話対応を削減できる
- 欠席者への宿題共有を自動化できる
- 宿題プリントの印刷業務を減らせる
- 生徒への宿題伝達漏れを防げる

といったメリットがあります。そのため、「どの業務をどの程度削減できるのか」を具体的に示し、現場の負担軽減につながることを伝える必要があります。

2. 生徒の個人情報を安全に管理する

スクールで利用するシステムでは、生徒や保護者の個人情報を扱うため、セキュリティ対策は不可欠です。そのため、

- 生徒IDでの管理
- アクセス権限の管理

などを行い、安全に利用できるシステムであることを示す必要があります。

3. コスト削減効果を可視化する

スクール側が導入を判断する上では、費用対効果も重要な要素です。

本アプリを導入することで、

- 宿題プリントの印刷コスト
- 月例単語テスト範囲表の印刷コスト
- 電話対応にかかる人件費

を削減できます。

## 8. このアプリの懸念点とその対策

### 懸念点①

- 何が問題になりそうか：全校生徒数千人規模で利用した場合、アプリ上で負荷や不具合が発生する可能性がある。
- なぜそう思うか：同時に集中することで、サーバー負荷の増加やレスポンス遅延が起こる可能性があるため。
- そのために考えている対策：段階的リリース（テスト運用→一部クラス→全校展開）を行う。
